## Шаг 7: Генерация ответов через Claude

### Созданные/изменённые файлы
- `internal/generator/claude.go` — `ClaudeGenerator` с методами `Generate` и `buildPrompt`

### Ключевые решения
- `anthropic.NewClient()` читает `ANTHROPIC_API_KEY` из окружения автоматически (стандарт SDK)
- `Generate`: таймаут 120 секунд (`context.WithTimeout`) — защита от зависания при медленном API
- Параметры генерации: `MaxTokens: 1024`, `Temperature: 0.3` (низкая температура → более детерминированные и точные ответы для RAG)
- **`buildPrompt`**: формирует промпт через `strings.Builder` с инструкциями на русском:
  - Отвечать только на основе предоставленного контекста
  - Сообщать, если информации недостаточно
  - Указывать источники (файл и раздел)
- Каждый контекстный чанк форматируется как `[Источник: X, Раздел: Y]\nтекст\n---`
- Подсказка в ошибке: `"Hint: check ANTHROPIC_API_KEY environment variable"`
- Модель задаётся через `CLAUDE_MODEL` (дефолт: `claude-sonnet-4-20250514` в `main.go`)

### Известные ограничения
- Промпт не учитывает лимит контекстного окна — при большом количестве контекстных чанков может превысить `max_tokens` модели
- `MaxTokens: 1024` ограничивает длину ответа; для сложных вопросов с развёрнутыми объяснениями может быть недостаточно
- Нет streaming — ответ приходит целиком, что ухудшает UX при длинных генерациях
