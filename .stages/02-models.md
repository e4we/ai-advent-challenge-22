## Шаг 2: Модели данных

### Созданные/изменённые файлы
- `internal/models/models.go` — структуры данных и функции генерации ID
- `internal/models/models_test.go` — 2 теста детерминированности и уникальности ID

### Ключевые решения
- `ChunkMetadata` хранит: `Source` (имя файла), `Title` (H1 заголовок), `Section` (H2/H3 заголовок), `ChunkID` (12-символьный hex), `Strategy` ("fixed"/"structural"), `CharCount`, `ChunkIndex`
- `Chunk` объединяет текст, метаданные и эмбеддинг (`[]float32`); поле `Embedding` помечено `json:"-"` — не сериализуется
- `SearchResult` — пара чанк + оценка релевантности (`float32`)
- `GenerateChunkID` — детерминированный 12-символьный hex из MD5(`source:strategy:index`), достаточно для логов и отладки
- `GeneratePointID` — детерминированный `uint64` из первых 8 байт MD5, используется как числовой ID точки в Qdrant (gRPC требует `uint64`)

### Известные ограничения
- MD5 не криптостойкий, но для детерминированной генерации ID коллизии практически исключены при типичных объёмах документов
- `GeneratePointID` возвращает `uint64`, что исключает использование UUID как ID точек Qdrant (выбрано сознательно для упрощения)
